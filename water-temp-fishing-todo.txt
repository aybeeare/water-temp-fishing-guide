1. GitHub — push the project if you haven't already
  git init
  git add .
  git commit -m "initial commit"
  # create repo on github.com, then:
  git remote add origin https://github.com/yourname/water-temp-fishing-guide.git
  git push -u origin main

  2. Google Places API key — before Railway
  console.cloud.google.com
  → New Project → Enable "Places API"
  → Credentials → Create API Key
  → Restrict key to "Places API" only (important)

  3. Railway
  railway.app → New Project → Deploy from GitHub repo
  Settings → Variables:
    ASSOCIATES_TRACKING_ID = watertemperat-20
    GOOGLE_PLACES_API_KEY  = your_key_here
    DAILY_PLACES_LIMIT     = 50
  Settings → Build Command:  python db/init_db.py
  Settings → Start Command:  uvicorn api.main:app --host 0.0.0.0 --port $PORT
  → Deploy → grab the public URL
  → Test: GET https://your-app.up.railway.app/health

  4. Lambda
  Zip and upload alexa_bridge/ per the earlier guide
  Environment variable:
    FASTAPI_URL = https://your-app.up.railway.app

  5. Connect Alexa ↔ Lambda
  Alexa Console → Build → Endpoint → paste Lambda ARN
  Copy Skill ID → Lambda → Triggers → Add Alexa Skills Kit → paste Skill ID

  6. Test in Alexa Console
  Test tab → "open fishing guide"
  "what's the water temp at Revere Beach"
